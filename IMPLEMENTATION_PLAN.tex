\section{Executive Summary}

CycleCare is a Kotlin/Jetpack Compose Android application for menstrual cycle tracking that already has a strong architectural foundation: MVVM + Clean Architecture, 7 Room database tables, AI-powered cycle predictions, biometric security, and data export.

This plan bridges the gap between CycleCare's current state and feature parity with \textbf{MyCalendar} (100M+ downloads, 4.9\,\(\bigstar\)). The roadmap is organized into \textbf{5 phases} spanning approximately \textbf{20 sprints (10 weeks)}, prioritized by user impact and technical dependency.

Database tables & 7 & 10 \\

Test coverage & Unit only & Unit + UI + Integration \\
Special modes & 0 & 3 (Pregnancy, Perimenopause, Abstinence) \\

\end{tcolorbox}

\section{Current State Analysis}

\subsection{Architecture (Existing — Solid)}
\begin{itemize}[leftmargin=1.5em]
  \item \textbf{Data Layer:} 7 Room entities, 7 DAOs, 5 repository implementations, DataExporter (CSV/JSON), notification workers
  \item \textbf{Domain Layer:} Clean models, repository interfaces, CyclePredictionEngine (weighted moving average + symptom correlation), ReminderScheduleEngine
  \item \textbf{Presentation Layer:} 4 screens (Calendar, DailyLog, Insights, Settings), ViewModels with StateFlow, Material Design 3 theme, PIN/biometric lock
  \item \textbf{DI:} Hilt modules (DatabaseModule, RepositoryModule, AppModule)
\end{itemize}

\subsection{What Works Well}
\begin{itemize}[leftmargin=1.5em]
  \item Clean Architecture separation is consistent
  \item Room schema covers core tracking (periods, daily logs, reminders, settings, health data, pregnancy, birth control)
  \item AI prediction engine with adaptive learning
  \item Biometric + PIN security already implemented
  \item Data export in CSV and JSON formats
\end{itemize}

\subsection{Critical Gaps}
\begin{enumerate}[leftmargin=1.5em]
  \item \textbf{Incomplete implementations:} PregnancyRepository, BirthControlRepository, HealthDataRepository return stubs
  \item \textbf{ReminderWorker:} Scheduled but missing actual notification dispatch
  \item \textbf{No cloud backup/restore} (Google Account sync)
  \item \textbf{Missing UI screens:} Onboarding, Pregnancy mode, Health dashboard, Partner sharing
  \item \textbf{No instrumentation tests;} only 2 unit test files
  \item \textbf{No special modes:} Pregnancy, Perimenopause, Abstinence
  \item \textbf{No wearable support} (Wear OS)
  \item \textbf{No educational content} or health condition tracking (PCOS, endometriosis, PMDD)
\end{enumerate}

\section{Privacy \& Security Considerations}

\begin{itemize}[leftmargin=1.5em]
  \item \textbf{Local-first architecture:} All health data stored in Room DB on device. Cloud backup is opt-in and encrypted.
  \item \textbf{Encryption:} AES-256-GCM for cloud backups. EncryptedSharedPreferences for sensitive settings.
  \item \textbf{No server telemetry on health data.} Analytics (if added) track only UI events, never cycle/symptom data.
  \item \textbf{Biometric gating:} Existing PIN/fingerprint lock extended to all new screens.
  \item \textbf{Partner sharing:} User explicitly controls what data categories are visible. Revocable anytime.
  \item \textbf{Data deletion:} Full account/data wipe from Settings with confirmation. Compliant with GDPR right to erasure.
  \item \textbf{Medical disclaimer:} App is not a medical device. All predictions are estimates. Prominent disclaimer in onboarding and settings.
\end{itemize}
